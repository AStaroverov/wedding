# Структура сайта — Double Alexander Project

Документ описывает, как текст приглашения превращается в секции сайта.
Для каждой секции: **назначение**, **содержимое** (дословно из текста), **компонент**, **источник данных**, **поведение / интеракция**, **состояние сейчас**.

Стиль и принципы (палитра, типографика, ритм скролла, анимации) — см. [`DESIGN.md`](./DESIGN.md). Здесь — только контент-структура.

---

## 0. Глобальные данные (единый источник правды)

Контент свадьбы выносится в `lib/tokens.ts` (`event`) и `lib/content.ts` (новый — длинные тексты, программа, дресс-код, transfer-инструкция, RSVP-конфиг). Любая секция читает из этих модулей — никакого хардкода в JSX.

**Что должно лежать в `event`:**
- `project`: `"Double Alexander Project"`
- `names`: `"Саша & Аля"`
- `date`: ISO `"2026-07-04T14:00:00+03:00"` (14:00 — церемония, основная точка)
- `dateLabel`: `"4 июля 2026"`
- `address`: `"Истринский район, СНТ «Рассвет», участок 104"`
- `coords`: `{ lat, lon }` — нужно уточнить, в `tokens.ts` сейчас старые координаты Москвы
- `mapUrl`: ссылка на «КАРТА» (Яндекс.Карты с пином)

**Что должно лежать в `content.ts`:**
- `greeting`: текст приветствия
- `travel`: массив способов добраться (car, taxi) — у каждого `title`, `body`, опц. `note`
- `program`: массив `{ time, title }`
- `dressCode`: `{ rule, allowedTones, mood, footwearNote }`
- `gifts`: текст про подарки
- `rsvp.questions`: декларативный список вопросов (см. §8)

## 1. Hero — обложка

**Назначение:** за 1 секунду сказать «это свадьба Саши и Али, 4 июля 2026». Эмоция, не информация.

**Содержимое из текста:**
- Надстрочник (caption): `Double Alexander Project`
- Главный заголовок (display): `Саша & Аля`
- Подпись: `4 июля 2026`

**Компонент:** `components/Hero.tsx` (уже есть).

**Как образована:**
- Полноэкранное фото (`/public/images/hero.png`), `objectFit: cover`, `priority`, `next/image`.
- Тёмный градиент-оверлей снизу для читаемости.
- Текст — нижняя треть, выровнен по центру, белый/cream.
- Скролл-индикатор: тонкая вертикальная линия, `opacity 0.5`.

**Что нужно добавить к текущей версии:**
- Над `h1` — caption `Double Alexander Project` (uppercase, letter-spacing 0.15em, opacity 0.85).
- В `event.names` подставить `"Саша & Аля"` вместо `"Имя & Имя"`.

**Высота:** `100svh`.

## 3. Program — программа дня

**Назначение:** показать таймлайн события одним взглядом.

**Содержимое из текста (структурировано):**

| Время | Событие |
|-------|---------|
| 13:00–13:30 | Сбор гостей |
| 14:00 | Церемония |
| 15:30 | Обед |
| 19:00 | Ужин |
| 22:00 | Окончание официальной части |

**Компонент:** `components/Program.tsx` — **новый, нужно создать.**

**Как образована:**
- Заголовок секции (caption): `Программа`
- Под ним — вертикальный таймлайн: слева монохромное время (`tabular-nums`, weight 500), справа название события.
- Между строками — тонкий разделитель `1px solid var(--line)`.
- Без иконок. Без картинок. Воздух важнее.
- Mobile: время и название в две строки в одной карточке.
- Появление: stagger fade-in (каждая строка с задержкой 80 ms).

**Данные:** `content.program` — массив `{ time: string, title: string }`.

**Высота:** `~80vh`.

**Содержимое из текста:**

- Ссылка-кнопка: `Открыть карту` (бывшая «КАРТА»)

**Компонент:** `components/Location.tsx` (уже есть, актуализировать адрес и координаты).

**Как образована:**

**Что нужно уточнить:**

## 5. Travel — как добраться

**Назначение:** снять у гостя тревогу про логистику. Это отдельная секция, **не вшивать в Location** — текст длинный и содержит call-to-action в RSVP («отметьтесь в опросе про парковку»).

**Содержимое из текста (дословно, разбито на 2 карточки):**

**Карточка 1 — На машине**
> На машине по Пятницкому шоссе. В дачном посёлке есть шлагбаум, мы сообщим как его можно будет открыть. Отметьтесь, пожалуйста, в опросе, если нужно парковочное место.

**Карточка 2 — На такси**
> На такси от МЦД Зеленоград. Стоимость — около 1000 ₽, время в пути — около получаса. В Зеленограде нет проблем с поиском машины, обратный поиск может занять 20–30 минут.

**Компонент:** `components/Travel.tsx` — **новый, нужно создать.**

**Как образована:**
- Заголовок (caption): `Как добраться`
- Две карточки в колонку (на desktop можно в две колонки, mobile — одна).
- Каждая карточка: иконка/эмодзи нейтральная или короткий лейбл (`Машина` / `Такси`), заголовок, абзац.
- Фон карточки: `--bg-soft`, радиус 12 px, padding 24–32.
- В первой карточке — ненавязчивая микро-ссылка-якорь `→ к опросу про парковку` (`#rsvp`).

**Данные:** `content.travel` — массив `{ id, title, body, link? }`.

**Высота:** `~80vh`.

## 6. DressCode — дресс-код

**Назначение:** дать чёткую визуальную инструкцию по одежде, снять страх «а вдруг я не так оделся».

**Содержимое из текста:**
- Правило: `all-white` — одежда белого цвета без ярких элементов и украшений.
- Допуски: низ пастельных тонов (светло-серый, бежевый).
- Тон: не строгий костюм/помпезное платье, не спортивная одежда — нарядно в нужных тонах.
- Рекомендации: удобная обувь + взять что-то тёплое на вечер.

**Компонент:** `components/DressCode.tsx` (уже есть, переписать контент).

**Как образована:**
1. Заголовок (caption): `Дресс-код`
2. Большой serif-заголовок: `all-white`
3. Палитра-кружки: 4–5 кругов 48–56 px — `#FFFFFF`, `#F5F1EA` (cream), `#E5DFD3` (beige), `#D9D6CF` (light grey). Под каждым — крошечная подпись с hex или названием тона.
4. Короткий абзац: «Нарядно, но без помпы. Удобная обувь и что-то тёплое на вечер.» (свести длинный текст к 1–2 фразам — детали можно убрать в `<details>` «Подробнее» либо оставить полностью, но визуально приглушённым `--ink-muted`).

**Данные:** `content.dressCode` — `{ rule, palette: [{hex,label}], notes }`.

**Высота:** `~50vh`.

## 8. RSVP — вопросы для гостей

**Назначение:** собрать ответы. Это **единственное действие** на сайте — оно повторяется sticky-кнопкой и/или находится в финале скролла.

**Содержимое из текста (вопросы дословно):**

| # | Поле | Тип | Опции |
|---|------|-----|-------|
| 1 | Имя и фамилия | text | required |
| 2 | Какие напитки вы предпочитаете? | single-choice | Больше алкогольные / 50 на 50 / Больше безалкогольные |
| 3 | Как вы будете добираться? | single-choice | Машина (я водитель) / Машина (я пассажир) / Такси |
| 4 | Под какую песню точно пойдёте танцевать? | text | optional |
| 5 | Есть ли у вас особые пожелания? | textarea | optional |

**Дополнительно (вытащить из текста Travel):**
- Если в вопросе 3 выбрано «Машина (я водитель)» — показать чекбокс «Нужно парковочное место».

**Компонент:** `components/Rsvp.tsx` (уже есть — нужно перестроить под новую структуру вопросов).

**Как образована:**
- Заголовок (caption): `Вопросы для гостей`
- Подзаголовок: одна строка с просьбой ответить до определённой даты (если нужна — уточнить).
- Поля идут вертикально, между ними `--space-6`.
- Радио-группы — пилюли (pill-toggle), не классические radio-кружки. Активная — заливка `--accent`, текст cream.
- Кнопка submit — pill, `--accent`, full-width на mobile.
- После отправки: inline-успех (fade-in `Спасибо! Ответ получен.`) без редиректа.
- Submit → `POST /api/rsvp` → Telegram (см. `lib/telegram.ts`).
- Валидация: React Hook Form + Zod. Имя — required, остальные радио — required, текстовые — optional.

**Данные:** `content.rsvp.questions` — декларативный список:

```ts
type RsvpQuestion =
  | { id: string; type: 'text' | 'textarea'; label: string; required: boolean }
  | { id: string; type: 'choice'; label: string; options: string[]; required: boolean };
```

Компонент рендерит форму **из массива**, чтобы новый вопрос = одна строчка в `content.ts`.

**Высота:** `~100vh` (форма заслуживает воздуха).

## 10. Итоговый порядок секций (`app/page.tsx`)

```
1. Hero          — обложка, имена, дата
2. Greeting      — приветствие
3. Program       — таймлайн дня         ← НОВАЯ
4. Location      — адрес + карта
5. Travel        — как добраться        ← НОВАЯ
6. DressCode     — all-white
7. Gifts         — про подарки          ← НОВАЯ
8. Rsvp          — форма вопросов
9. Footer
```

Порядок отражает путь читателя: **«куда зову» → «что будет» → «как доехать» → «как выглядеть» → «что взять (ничего)» → «ответь мне»**.

## 11. Что нужно сделать после утверждения структуры

1. Создать `lib/content.ts` со всеми текстами и массивами по схемам выше.
2. Обновить `lib/tokens.ts` — реальные координаты, `mapUrl`, `project`, актуальные `names`/`dateLabel`.
3. Создать компоненты: `Program.tsx`, `Travel.tsx`, `Gifts.tsx`.
4. Переписать содержимое: `Hero.tsx` (caption), `Greeting.tsx`, `DressCode.tsx`, `Location.tsx`, `Rsvp.tsx` (новые вопросы).
5. Обновить порядок в `app/page.tsx`.
6. Уточнить недостающие данные:
   - Точные координаты участка + ссылка на Яндекс.Карты.
   - Телефон/контакт организатора для футера.
   - Нужен ли дедлайн ответа в RSVP.
   - Финальное фото для Hero.

## 12. Что НЕ делать (зеркало DESIGN.md §12 для нашего случая)

- Не вшивать тексты в JSX — всё через `content.ts`.
- Не делать секцию «История о нас» / «Таймер» — в исходном тексте их нет, не выдумывать.
- Не делать секцию «Подарки» в виде wishlist/реквизитов — в тексте чёткое «никаких вещей».
- Не делать accordion для всей программы — она короткая, открыта по умолчанию.
- Не делать в Travel ссылку на сервисы такси — в тексте только описательно.
