# Сайт-визитка: принципы и архитектура

Референс: https://daniilelzara.ru/ — свадебное приглашение-одностраничник на платформе Eventrix.pro (Next.js + Chakra UI).

> Замечание: оригинал — SPA с lazy-рендером, headless-снапшоты не дали полной картины. Документ собран из доступной информации (мета-теги, OG-превью, стек, типичный жанр long-scroll wedding/personal landing) и описывает **принципы**, по которым такие сайты строятся, чтобы повторить ощущение, а не скопировать поштучно.

---

## 1. Жанр и цель

**Тип:** persona-landing / one-page invitation / digital business card.
**Цель:** за 30 секунд скролла дать гостю/посетителю эмоциональное впечатление о людях/событии и побудить к одному явному действию (RSVP, контакт, бронь).

Один экран = одна мысль. Длинный вертикальный скролл — это сторителлинг, а не каталог.

---

## 2. Принципы дизайна

### 2.1. Эмоция важнее информации
- Главный носитель смысла — **большое фото** и **типографика**, не текст.
- Текст краткий: имена, дата, 1–2 фразы на блок.
- Каждая секция оставляет «вдох» (воздух/паддинги > контента).

### 2.2. Минимализм + единый акцент
- Фон: off-white / cream (`#FAF7F2`, `#F5F1EA`) или soft-beige.
- Один акцентный цвет (terracotta / sage / dusty-rose / olive). Не два.
- Тени почти не используются. Разделение — воздухом и контрастом фото.

### 2.3. Mobile-first
Реальные просмотры — с телефона из мессенджера. Дизайн строится в колонке 360–414 px, десктоп = центрированная колонка ≤ 480–600 px на светлом фоне.

### 2.4. Тихий ритм скролла
- Высота большинства секций ≈ 100vh или чуть меньше.
- Между секциями — пауза (декоративный divider: тонкая линия, маленькая иконка, орнамент).
- Анимации появления **на въезде** (fade + translateY 16–24 px, 600–800 ms, ease-out). Никаких пружин.

### 2.5. Один тип действия
RSVP-кнопка/контакт/CTA — sticky либо повторяется в 2–3 местах. Без меню навигации (или скрытое burger). Скролл — сам себе навигация.

## 3. Цвет, типографика, токены

### Палитра (пример, основан на cream + sage + ink)
```
--bg-primary:    #FAF7F2  /* основной фон */
--bg-soft:       #F0EBE2  /* секции-вставки */
--ink:           #2B2A28  /* основной текст */
--ink-muted:     #6B6862  /* подписи, мета */
--accent:        #8A9A7B  /* sage / акцент */
--accent-soft:   #DDE3D4
--line:          #E5DFD3  /* разделители */
```

### Типографика
- **Display (имена, даты):** serif с характером — Cormorant Garamond, Playfair Display, EB Garamond. Размер 48–96 px desktop / 36–56 px mobile.
- **Body:** humanist sans — Inter, Manrope, Geist. 16/24, weight 400.
- **Caption:** uppercase, letter-spacing 0.15em, weight 500, размер 12–13 px.
- **Числа в таймере / даты:** tabular-nums.

Паттерн: один serif + один sans. Не больше двух семейств.

### Spacing
4-пиксельная шкала: 4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96 / 128.
Секционный `padding-block`: 96–128 desktop, 64–80 mobile.

### Радиусы и линии
- Кнопки/карточки: `border-radius: 999px` (pill) или `12px` (мягкий прямоугольник).
- Линии-разделители: `1px solid var(--line)`, длина 40–80 px, по центру.

## 4. Структура секций (long-scroll сценарий)

Порядок блоков для wedding/persona invitation:

| # | Секция | Содержимое | Высота |
|---|--------|------------|--------|
| 1 | **Hero / обложка** | большое фото (или фото-фон) + имена + дата + город | 100vh |
| 2 | **Приветствие** | 1–2 предложения, обращение к гостю | ~60vh |
| 3 | **Таймер** | обратный отсчёт до даты (DD:HH:MM:SS) | ~40vh |
| 4 | **История / о нас** | 1–2 абзаца + 1–2 фото | ~80vh |
| 5 | **Программа дня** | таймлайн (15:00 регистрация → 16:00 церемония → 19:00 банкет) | ~80vh |
| 6 | **Локация** | адрес + карта (Яндекс/Google) + кнопка «как добраться» | ~70vh |
| 7 | **Дресс-код** | палитра (3–5 цветовых кружков) + 1 фраза | ~40vh |
| 8 | **Пожелания / подарки** | вместо цветов — куда / что | ~40vh |
| 9 | **RSVP-форма** | имя, придёте ли, +1, аллергии, кнопка | ~80vh |
| 10 | **Футер** | контакты организатора, соц-сети, спасибо | ~30vh |

Для **persona-визитки** (не свадьба) тот же скелет:
hero → tagline → about → проекты/услуги (горизонт. карусель или вертикальные карточки) → отзывы/цифры → contact form → footer.

## 5. Hero-секция (детально)

```
┌─────────────────────────┐
│                         │
│    [фото на весь экран] │
│       overlay 30% dark  │
│                         │
│        ИМЯ & ИМЯ        │  ← serif, 72px
│                         │
│      10.05.2026         │  ← caption
│       Москва            │
│                         │
│         ↓               │  ← scroll hint
└─────────────────────────┘
```

- Фото высокого разрешения, **subtle parallax** (translateY ~10% на скролл).
- Текст по центру или нижняя треть, белый/cream.
- Скролл-индикатор: тонкая стрелка/линия, opacity ≤ 0.6.

## 6. Анимации и микро-взаимодействия

| Триггер | Эффект | Параметры |
|---------|--------|-----------|
| Въезд секции в viewport | fade + translateY | 16–24 px → 0, 600–800 ms, ease-out, threshold 30% |
| Hover на кнопке | scale + цвет | 1.0 → 1.03, 200 ms |
| Hover на фото | zoom внутри маски | scale 1.05, 400 ms |
| Тик таймера | tabular цифры обновляются | без анимации, монотонно |
| Открытие RSVP | accordion / smooth scroll | 400 ms ease-in-out |
| Загрузка | минимальный сплеш | «Загрузка приглашения…» fade-in/out |

Библиотека: **Framer Motion** (`whileInView`, `viewport={{ once: true }}`). Никакого jQuery/AOS.

## 7. Технический стек (предлагаемый)

Соответствует референсу (Next.js обнаружен в `_next/static`):

- **Framework:** Next.js 14+ (App Router) — SEO, OG-картинки на лету через `@vercel/og`.
- **UI:** Chakra UI **или** Tailwind CSS + shadcn/ui. Один из них.
- **Анимации:** Framer Motion.
- **Карта:** Яндекс.Карты JS API (RU-аудитория) или mapbox-gl.
- **Формы (RSVP):** React Hook Form + Zod валидация. Бэкенд — Telegram-бот webhook / Google Sheets через Apps Script / Supabase Functions. Не делать полноценный backend — одна форма не стоит того.
- **Изображения:** `next/image`, AVIF + WebP, `sizes` обязательно. Hero-фото предзагружено через `<link rel="preload" as="image">`.
- **Шрифты:** `next/font` (self-hosted, без FOUT).
- **Аналитика:** Yandex.Metrika (как в референсе) или Plausible.
- **Хостинг:** Vercel / Netlify / Cloudflare Pages.

## 8. Производительность и SEO

- **LCP < 2.5 s.** Hero-фото — приоритет, остальные lazy.
- **CLS = 0.** Фиксированные `aspect-ratio` для всех картинок.
- Шрифты — `font-display: swap`, subsetted (только нужные кириллица/латиница).
- OG-картинка генерируется динамически (`/api/og/...png` 1200×630 — в референсе 500×500).
- `<meta>` ru-локаль, `description`, `og:*`, `twitter:card=summary_large_image`.
- Один HTML, **без роутов**. URL-якоря (`#rsvp`, `#program`) для шаринга секций.

---

## 9. Доступность

- Контраст текст/фон ≥ 4.5:1 (тёмный текст на cream — ок; белый текст на фото-героя — добавить overlay).
- `prefers-reduced-motion`: отключать `translateY`/parallax, оставлять fade.
- Все интерактивные элементы — `<button>`/`<a>` с `:focus-visible` (видимый outline 2 px accent).
- Карта — иметь текстовую альтернативу (адрес рядом).
- Форма — `aria-describedby` для ошибок, `aria-live="polite"` для подтверждения.

## 10. Контент-чеклист (что собрать ДО кода)

- [ ] 6–10 финальных фото (горизонт + вертикаль), retouched, > 2000 px по длинной стороне.
- [ ] Имена, дата, время, точный адрес с координатами.
- [ ] Тексты: приветствие (≤ 30 слов), история (≤ 100 слов), дресс-код (≤ 15 слов).
- [ ] Программа дня — список «время → событие».
- [ ] Палитра дресс-кода — 3–5 hex-кодов.
- [ ] RSVP-поля: что точно нужно (имя, +1, питание, трансфер?).
- [ ] Куда уходят ответы RSVP (Telegram-чат / таблица).
- [ ] Домен и DNS.

## 12. Что НЕ делать

- Не вставлять фоновую музыку с автоплеем.
- Не делать слайдер фото, который надо листать руками — ленивый скролл-галерея лучше.
- Не использовать > 2 шрифтов и > 1 акцентного цвета.
- Не верстать на «секциях по 100vh» жёстко — на низких экранах ломается, использовать `min-height: 100svh; padding-block: clamp(...)`.
- Не хранить RSVP в localStorage — теряется. Сразу на сервер.
- Не блокировать загрузку фоновым видео > 2 МБ.
- Не делать `position: fixed` хедер на мобильном — крадёт высоту.
